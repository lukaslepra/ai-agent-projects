```

  同一个核心 Prompt 跑3个候选模型，比质量再最终决定
```

### 6. 微调基础（面试必问）

**一句话判断是否需要微调：**

> 如果 Prompt 调了3遍还是不稳定，考虑 RAG；RAG 也搞不定，再考虑微调。

**三种方案对比：**

| 方案 | 解决什么问题 | 成本 | 什么时候用 |
|------|-------------|------|-----------|
| Prompt Engineering | 调整模型行为和输出风格 | 极低 | 80%的场景 |
| RAG（检索增强） | 注入私有/实时知识 | 中 | 需要私有文档 |
| SFT微调 | 固定输出格式/模仿特定风格 | 高 | 格式要求极严格 |
| LoRA | SFT的轻量版 | 中高 | 有GPU，数据充足 |

**RLHF（面试高频）一句话解释：**
> 让真人给模型回答打分，用分数训练一个"偏好评分模型"，再用它引导基础模型往人类更喜欢的方向迭代优化。GPT-5.5、Claude Opus 4.8 都用了这个技术。

RAG（检索增强生成）是一种将信息检索与大语言模型生成能力结合的技术，模型在生成回答前会先从外部知识库中检索相关内容，再以此为上下文生成更准确的答案。它解决了大模型"知识截止日期"和"幻觉"两大核心问题，让模型能够访问实时或私有数据。与纯参数化记忆相比，RAG的知识更易于更新和验证。

**实际应用场景：** 企业内部知识库问答——员工提问"年假政策是什么？"时，系统先从文档库检索相关段落，再由模型基于这些段落生成准确回答，确保信息可追溯。

Token：输入=4482，输出=333

**【Claude Haiku 4.5】**

RAG（检索增强生成）是一种技术，它先从外部知识库中检索相关信息，然后将这些信息作为上下文输入给大语言模型，最后由模型生成更准确、更有根据的回答。这种方法结合了检索系统的精确性和生成模型的灵活性，能够减少幻觉问题，提供更可信的输出。

**实际应用场景：** 企业客服系统——客户提问时，系统先从文档库检索相关内容，再由模型生成个性化回答，确保准确且符合企业信息。

1. **豆包和千问分别最适合什么场景？背后的逻辑是什么（提示：想想各自的母公司生态）？**

豆包更适用于文本类的，自媒体的一些场景
千问最适用于RAG知识库之类的
千问是阿里巴巴旗下的大模型，更适用于专业场景
豆包是字节的，更适用于文娱类型场景

2. **智谱 GLM-5 和 DeepSeek V4-Pro 都是国产模型，你会怎么在这两个之间选择？**

选 GLM-5 的场景：需要长程复杂 Agent 任务（支持单次持续工作8小时）、预算有限但需要 Claude Opus 4.6 级综合能力、有私有化部署需求的企业场景。
选 DeepSeek V4-Pro 的场景：算法推理和代码竞赛类任务（LiveCodeBench 得分 93.5，高于 GLM-5）、极致低价的大批量推理（¥13/¥25 per M token）、1M 超长上下文场景。
核心区分逻辑：GLM-5 胜在「综合能力 + 长程 Agent」，DeepSeek V4-Pro 胜在「算法推理 + 极致性价比」。代码工程不是 GLM-5 的核心优势，不要搞混。

3. **Claude Opus 4.7 是SWE-bench第一，但为什么实际项目里几乎没人用它做主力模型？**

太贵了！！还有就是国内不合规！

4. **你的项目三（合同风险审查Agent）最终选哪个模型？给出完整理由（至少3条）。**

初期验证阶段选 DeepSeek V4-Flash，理由：
1. 价格最低（¥1/¥2 per M token），每天1000次合同审查月费用仅 ¥330，是 Claude Sonnet 的 1/30，适合 MVP 阶段快速跑通流程
2. 中文写作胜率 62.7%，合同文本是中文场景，能力完全够用
3. 1M 超长上下文，即便是超长合同也不会截断，不需要额外处理分块

风险应对策略：先用 Flash 在测试集上跑准确率，如果风险识别漏判率超过阈值，再切换到 DeepSeek V4-Pro 或千问 Qwen3.7-Max 对比，逐步调优。不一开始就用贵的模型——用最低成本验证，数据说话再升级，这是工程师思维。

5. **用一句话解释 RLHF，不能用"偏好"和"强化学习"这两个词。**（锻炼用自己的话表达）

就是对模型输出进行评分后训练一个奖励模型，奖励模型会让基座模型输出进行评定，让基座模型输出更倾向于人类喜好的内容，然后不断地对基座模型进行迭代升级优化
